# Supabase 작업 규칙

## DB 초기화 파일 유지보수

`migrations/` 에 새 마이그레이션 파일이 추가되거나 기존 파일이 수정될 때마다
아래 두 파일을 반드시 함께 갱신해야 한다.

### reset_db.sql (스키마 + 데이터 전체 재생성)

변경이 필요한 경우:
- 테이블 추가 / 삭제 / 컬럼 변경
- 인덱스 / 제약 조건 변경
- RLS 정책 추가 / 수정 / 삭제
- 헬퍼 함수 추가 / 수정
- 트리거 추가 / 수정
- 초기 seed 데이터 변경 (plan_limits 등)

갱신 방법:
- STEP 1 DROP 섹션: 새 테이블·함수가 있으면 DROP 대상에 추가
- STEP 2 테이블 생성: 새 컬럼·제약·테이블을 현재 최종 상태로 반영
- STEP 3 인덱스: 신규 인덱스 추가
- STEP 5 헬퍼 함수: 함수 내용 최신화
- STEP 6 RLS 정책: 정책 추가·수정·삭제 반영
- STEP 7 트리거: 트리거 함수 내용 최신화
- STEP 9 초기 데이터: seed 데이터 변경 반영

### reset_data.sql (데이터만 삭제, 스키마 유지)

변경이 필요한 경우:
- 새 테이블이 추가되어 TRUNCATE 대상에 포함해야 할 때
- 테이블이 삭제되어 TRUNCATE 대상에서 제거해야 할 때
- 초기 seed 데이터가 변경될 때 (plan_limits 등)

갱신 방법:
- STEP 2 TRUNCATE 목록: 새 테이블을 FK 자식→부모 순서로 추가
- STEP 3 초기 데이터: seed 데이터 변경 반영

## 규칙 요약

1. 마이그레이션 작업이 완료되면 두 파일 갱신을 체크한다.
2. 갱신 시 전체 마이그레이션 이력(001~최신)의 누적 최종 상태를 기준으로 작성한다.
   (중간 단계가 아닌 현재 최종 스키마를 반영)
3. reset_db.sql 파일 상단의 "기준 마이그레이션" 주석을 최신 번호로 업데이트한다.
