## 1. 셋업 위자드 AI 설정 어시스턴트 (`SetupAssistant.tsx`)

배경: 신규 조직 셋업 시 역할·커스텀필드·휴무요일·정원을 단계별로 하나씩 입력해야 했음. 위자드 1단계에서 운영 방식을 자연어로 설명하면 AI(Claude, Edge Function `ai-parse`)가 역할/커스텀필드/휴무요일/정원을 제안하고, 확인 후 각 단계에 자동 반영하도록 개선.

- [ ] 위자드 1단계에 "AI로 빠르게 시작하기" 진입 버튼이 보임
- [ ] 버튼을 누르면 운영 방식을 입력하는 텍스트영역이 열림
- [ ] "역할은 팀장, 강사, 봉사자가 있고 평일 9시~18시 운영, 일요일은 휴무예요. 커스텀필드로 연락처와 특이사항이 필요해요." 같은 문장을 입력 후 "분석하기" 클릭 시 역할 3개·커스텀필드 2개·휴무요일(일)이 제안됨
- [ ] "적용하기" 클릭 시 역할 단계·커스텀필드 단계에 반영됨 (각 단계 화면에서 실제로 채워졌는지 이동해서 확인)
- [ ] 휴무 요일·정원은 3단계(슬롯) 저장 후에 반영된다는 안내 문구대로, 3단계 진입/저장 시 실제로 반영됨
- [ ] 적용 후에도 각 단계에서 자유롭게 값을 수정할 수 있음 (AI 제안이 값을 잠그지 않음)
- [ ] 구체적인 내용이 없는 문장(예: "그냥 평범하게 운영해요") 입력 시 "설명에서 구체적인 내용을 찾지 못했어요" 안내가 뜨고 에러 없이 처리됨
- [ ] 네트워크 실패/서버 에러 시 에러 메시지가 폼 위에 노출되고, 재시도(다시 분석하기) 가능함
- [ ] "닫기"로 어시스턴트를 접었다가 다시 열어도 이전 입력 상태와 무관하게 정상 동작함

## 2. 헤더 자연어 빠른예약 (`QuickBookingModal.tsx`)

배경: 예약 등록/변경/취소를 매번 폼으로 입력해야 했음. 헤더에 관리자 전용 AI 버튼(✨)을 추가해 "홍길동님 이번주 일요일 9시 예약" 같은 자연어로 등록·변경·삭제를 처리하고, 파싱 결과를 확인·수정 후 확정하도록 개선.

- [ ] 조직 관리자로 로그인 시 헤더에 ✨ 버튼이 보임 (일반 회원/비로그인 시 보이지 않음)
- [ ] **등록(create)** — "홍길동님 이번주 일요일 9시부터 12시까지" 입력 시 대상자·날짜·복수 시간대가 올바르게 파싱됨 (실제 회원 명단에 있는 이름으로 테스트)
- [ ] 등록 시 이름이 정확히 일치하지 않아도(오타·부분일치) 유사 후보가 드롭다운에 뜨고 선택 가능함 (`fuzzyMatchName`)
- [ ] 등록 시 이미 같은 회원·시간대에 배정이 있으면 "중복 N건 제외"로 안내되고 나머지만 등록됨
- [ ] **반복 등록** — "매주 화요일 7월부터 9월까지 강습 등록" 처럼 반복 패턴 입력 시 요일·기간이 파싱되고, 휴무일/휴관일은 등록 대상에서 자동 제외됨
- [ ] **변경(update)** — "홍길동님 7월 20일 9시 예약을 10시로 변경" 입력 시 기존 예약이 후보로 뜨고, 새 시간으로 변경됨
- [ ] **삭제(delete)** — "홍길동님 7월 전체 15시 일정 삭제" 처럼 기간(range) 지정 삭제 시 해당 기간의 대상 후보가 모두 뜨고, 선택 해제 후 일부만 삭제도 가능함
- [ ] 변경/삭제 대상 후보가 여러 건일 때 "전체 선택"/"전체 해제"가 정상 동작함
- [ ] AI가 계산한 요일과 실제 날짜의 요일이 다르면 경고 문구가 뜸 (`weekdayMismatch`)
- [ ] 필수 커스텀 필드가 있는 조직에서 등록 시, 필수값 미입력이면 등록 버튼이 비활성화됨
- [ ] 비회원(방문자 예약) 모드 조직에서는 회원 선택 대신 이름 직접입력 필드가 뜨고 정상 등록됨
- [ ] "N시"와 "N일" 표현이 섞인 문장(예: "15시에 20일 예약")에서 시간·날짜가 서로 바뀌지 않고 올바르게 파싱됨
- [ ] 처리 완료 후 "N건 등록/변경/삭제했어요" 메시지가 뜨고, 모달을 닫으면 스케줄 화면(월/주/일 뷰)에 실제로 반영되어 보임
