# [제 2편] "이번 주 수요일은 쉬어요"를 코드로 — 반복 규칙과 날짜 예외 처리

1편에서 멀티테넌트 구조와 세 가지 운영 모드를 다뤘다면, 이번 편의 주제는 훨씬 현실적인 문제입니다. **"우리는 화·목만 운영하는데, 다음 달 셋째 주 화요일은 워크숍 때문에 쉬어요."** 이 한 문장을 시스템이 어떻게 이해하게 만들 것인가, 하는 문제입니다.

반복되는 규칙과 그 규칙을 깨는 예외. 캘린더를 다뤄본 개발자라면 누구나 이 둘의 관계에서 한 번쯤 골머리를 앓아봤을 겁니다.

---

## 1. 실사용 시나리오: 반복 규칙만으로는 안 되는 이유

### 시나리오: 격일 근무하는 필라테스 강사팀

강사 3명이 월·수·금만 수업하는 스튜디오를 예로 들어보겠습니다. 처음 조직을 설정할 때는 간단합니다. "월·수·금 운영"이라는 규칙 하나면 1년치 캘린더가 알아서 채워집니다.

문제는 그다음부터입니다.

- 추석 연휴가 낀 주는 원래 수요일 운영일이지만 쉬어야 한다.
- 반대로 원래 미운영일인 화요일에, 특강 이벤트 때문에 임시로 문을 연다.
- 점심시간(12~13시)은 매일 공통으로 휴게시간이라 배정이 불가능해야 한다.

셋 다 "반복 규칙"이 아니라 "그날 하루만 다른 규칙"입니다. 만약 이걸 반영하겠다고 매번 요일 규칙 자체를 수정한다면, 추석 다음 주부터는 다시 원래대로 되돌려야 하는 번거로움이 생깁니다. DTS는 이 둘을 처음부터 분리된 개념으로 설계했습니다.

* **스케줄 규칙(`schedule_rules`)**: "매주 반복되는" 요일별·시간대별 운영 여부
* **날짜 오버라이드(`date_overrides`)**: "이 날짜 하나만" 규칙을 뒤집는 예외

우선순위는 항상 날짜 오버라이드가 이깁니다. 규칙이 "운영"이라고 해도 그날 오버라이드가 "휴관"이면 닫힙니다.

> **📸 [이미지 삽입 구간: 월간 캘린더에서 반복 운영일과 휴관일이 함께 보이는 화면]**
> *설명: 월·수·금은 정상 배경, 추석 연휴가 낀 수요일 한 칸만 휴관 줄무늬로 표시되는 실제 캘린더 스크린샷. "규칙"과 "예외"가 한 화면에서 어떻게 다르게 보이는지 한눈에 보여주는 용도.*

## 2. 관리자 조작 가이드: 규칙 템플릿으로 5초 만에 시작하기

매주 요일을 하나하나 클릭해서 켜고 끄는 건 번거롭습니다. 그래서 자주 쓰이는 조합을 템플릿으로 미리 만들어뒀습니다.

> **📸 [이미지 삽입 구간: 관리자 콘솔 - 스케줄규칙 탭, 템플릿 선택 드롭다운]**
> *설명: "연중무휴 / 평일만(월-금) / 평일+토(월-토) / 주말만(토·일) / 월·수·금 / 화·목" 6개 템플릿 버튼이 나열된 화면.*

1. 관리자 콘솔 → **스케줄규칙** 탭으로 이동합니다.
2. 상단의 템플릿 중 조직 운영 방식과 가장 가까운 것을 고릅니다. 예를 들어 필라테스 스튜디오라면 **"월·수·금"**을 선택합니다.
3. 템플릿을 적용한 뒤에도 특정 요일·시간대만 개별적으로 켜고 끌 수 있습니다. 격자 형태의 표에서 셀을 클릭하면 운영/미운영이 토글됩니다.
4. **공휴일 포함 여부**도 템플릿마다 다릅니다. "연중무휴"만 공휴일을 포함하고, 나머지 템플릿은 기본적으로 일요일과 공휴일을 미운영으로 취급합니다.

특정 날짜 하나만 바꾸고 싶다면 규칙 탭이 아니라 **날짜설정** 탭으로 넘어갑니다.

> **📸 [이미지 삽입 구간: 관리자 콘솔 - 날짜설정 탭, 캘린더에서 특정 날짜 클릭 후 팝업]**
> *설명: 날짜를 클릭하면 "휴관일로 지정" / "특별 운영일로 지정" 두 버튼이 뜨는 팝업 화면.*

5. **날짜설정** 탭에서 예외를 적용할 날짜를 클릭합니다.
6. 원래 운영일인데 하루 쉬려면 **휴관일 지정**을, 원래 미운영일인데 하루 열려면 **특별 운영일 지정**을 선택합니다.
7. 저장하면 그 날짜에 한해서만 규칙이 뒤집힙니다. 다른 날짜는 전혀 영향받지 않습니다.

## 3. 개발자 비하인드: 우선순위를 결정하는 한 함수

이 모든 판단은 `getCellState()`라는 함수 하나에 모여 있습니다. 특정 날짜·시간대 셀이 "휴관"인지, "휴게"인지, "정상 운영"인지를 계산하는 순수 함수입니다.

```typescript
// src/utils/cellState.ts
const override = dateOverrides.find(d => d.date === dateStr)
const rule = scheduleRules.find(r => r.day_of_week === dayOfWeek && r.time_slot === timeSlot)

// 공휴일 판정: 오버라이드 명시, 또는 일요일/공휴일인데 운영 규칙이 없는 경우
const isHoliday = override?.is_holiday === true
  || (dayOfWeek === 0 && rule?.is_open !== true)
  || (rule?.is_open !== true && override === undefined && isKoreanHoliday(dateStr))

// 휴게시간: 공휴일이 아닌 날에 규칙 자체가 닫혀 있는 경우
const isClosedByRule = rule ? !rule.is_open : true
const isBreaktime = isClosedByRule && !isHoliday && override === undefined
```

여기서 핵심은 순서입니다. `override`(날짜 예외)를 가장 먼저 찾아두고, 그다음 `rule`(요일 규칙)을 찾습니다. 이후 판정 로직 어디에서도 `override !== undefined`인 경우엔 규칙 대신 오버라이드 값을 우선 사용합니다.

흥미로운 지점은 공휴일 판정 조건입니다. `rule?.is_open === true`인 요일·시간대라면, 그날이 일요일이거나 공식 공휴일이어도 **휴관으로 처리하지 않습니다.** "연중무휴" 템플릿을 쓰는 조직(예: 방문 요양 센터)이 명절에도 정상 운영해야 하는 현실을 반영한 예외입니다. 반대로 평일만 운영하는 조직은 애초에 일요일 규칙이 `is_open: false`이므로 자연스럽게 휴관으로 떨어집니다.

`isKoreanHoliday()` 함수는 매년 바뀌는 음력 공휴일(설날, 추석)까지 계산해야 하는데, 이 부분은 별도 유틸(`koreanHolidays.ts`)에서 연도별 공휴일 목록을 생성해 처리합니다.

## 4. Troubleshooting: "연중무휴인데 왜 일요일에 줄무늬가 안 생기죠?"

바로 위에서 고친 `isHoliday` 조건을 실제로 적용하고 나니, 이번엔 다른 지점에서 아쉬운 점이 눈에 띄었습니다. 연중무휴로 설정한 요양 센터라면, 캘린더 화면에서 일요일과 공휴일에도 정상 운영일과 똑같은 흰 배경이 보입니다. 배정은 정상적으로 가능하지만, "오늘이 일요일/공휴일이다"라는 시각적 구분 자체가 사라져버린 셈입니다.

### 🔴 문제 상황

배경색을 결정하는 로직이 `cs.isHoliday` 플래그만 보고 있었는데, 위에서 봤듯 연중무휴 조직은 `rule.is_open === true`라서 애초에 `isHoliday`가 `false`로 계산됩니다. 배정 가능 여부(비즈니스 로직)는 정확했지만, **일요일·공휴일임을 시각적으로 알려주는 줄무늬 배경**이 통째로 사라져버린 것이었죠. 코드는 맞았는데 사용자 기대와 어긋난 케이스입니다.

### 🟢 해결 과정

"휴관 여부"와 "오늘이 일요일/공휴일인지 여부"는 서로 다른 질문이라는 걸 명확히 하는 게 핵심이었습니다. `isHoliday`는 배정 가능 여부를 결정하는 비즈니스 플래그로 그대로 두고, 화면 렌더링 쪽에만 "오늘 요일이 일요일 또는 공휴일인가"를 별도로 계산하는 얕은 체크를 추가했습니다.

```typescript
// 개념적 흐름 — 운영 가능 여부(isHoliday)와 무관하게 "일요일/공휴일" 자체를 렌더링 쪽에서 따로 판단
const isSundayOrHoliday = dayOfWeek === 0 || isKoreanHoliday(dateStr)
```

이렇게 하면 연중무휴 조직도 일요일엔 옅은 줄무늬가 깔리되, 그 셀은 여전히 클릭해서 배정을 등록할 수 있는 상태로 유지됩니다. "쉬는 날처럼 보이지만 실제로는 운영 중"이라는, 얼핏 모순돼 보이는 요구사항이 이렇게 풀렸습니다.

> **📸 [이미지 삽입 구간: 수정 전/후 비교 — 연중무휴 조직의 일요일 셀]**
> *설명: 왼쪽은 수정 전(일요일도 평일과 똑같은 흰 배경), 오른쪽은 수정 후(옅은 줄무늬 배경 + 클릭 시 정상적으로 배정 등록 팝업이 뜨는 모습)를 나란히 배치한 비교 스크린샷.*

이 수정은 월간 뷰뿐 아니라 주간 뷰, 그리고 최근 추가된 일자별 요약 뷰에도 동일하게 적용해야 했습니다. 뷰마다 렌더링 코드가 따로 있다 보니, 한 곳만 고치고 다른 곳을 빠뜨리는 실수가 실제로 한 번 있었습니다. 그 이후로 스케줄 표시에 영향을 주는 변경은 반드시 월/주/일 뷰 전체를 함께 점검하는 걸 원칙으로 세웠습니다.
