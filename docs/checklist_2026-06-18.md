## 1. 모바일/태블릿 가로모드(landscape) 허용

배경: PWA manifest에 `orientation: "portrait" / "portrait-primary"`가 고정되어 있어,
"홈 화면에 추가"로 설치한 PWA(standalone 모드)에서 기기를 가로로 돌려도 화면이 회전하지 않는 문제가 있었다.
`vite.config.ts`(VitePWA manifest)와 `public/manifest.json`에서 해당 `orientation` 속성을 제거했다.

- [ ] (기존에 홈 화면에 설치된 앱이 있다면) 기존 PWA 아이콘 삭제 후 재설치 — manifest는 설치 시점에 캐시되므로 재설치해야 변경이 반영됨
- [ ] 모바일(휴대폰)에서 PWA를 홈 화면에 새로 설치 → 기기를 가로로 돌렸을 때 화면이 가로로 회전하는지 확인
- [ ] 태블릿에서 PWA를 홈 화면에 새로 설치 → 가로/세로 전환 시 모두 정상 회전하는지 확인
- [ ] 일반 모바일 브라우저(설치 없이 탭으로 접속)에서도 가로모드 전환이 정상인지 확인 (원래도 영향 없었어야 하는 경로)
- [ ] 가로모드 상태에서 월/주/일/모바일 스케줄 뷰 레이아웃이 깨지지 않는지 확인 (현재 가로모드 전용 CSS는 없으므로 기존 반응형 레이아웃 그대로 적용됨)

## 2. 모바일/태블릿 가로모드 달력 상하좌우 여백 통일

배경: 달력 카드의 외곽 여백(`<main>` 패딩, 카드 테두리/그림자, 헤더 패딩, 콘텐츠 패딩 등)이
Tailwind `sm:`(가로폭 640px 이상) 기준으로만 적용되어 있었다. 휴대폰을 가로로 돌리면
가로폭이 640px를 넘는 경우가 많아 세로모드에는 없던 여백/테두리/그림자가 갑자기 생기는 문제가 있었다.
`src/index.css`에 `framed:` 커스텀 variant(`min-width: 640px` **그리고** `min-height: 640px`)를 추가하고,
`SchedulePage.tsx`/`WeekGrid.tsx`/`DayView.tsx`의 외곽 여백 관련 `sm:` 클래스를 `framed:`로 교체했다.
(휴대폰은 가로로 돌려도 짧은 변의 길이=세로모드 폭이 640px를 넘지 않으므로 계속 여백 없는 레이아웃을 유지한다.)

- [ ] 휴대폰 세로모드 → 가로모드 전환 시 달력 카드 테두리/그림자/패딩이 추가로 생기지 않고 세로모드와 동일하게 여백 없이 표시되는지 확인 (월/주/일 뷰 모두)
- [ ] 태블릿 세로모드 ↔ 가로모드 전환 시에도 여백 표시가 일관되게 유지되는지 확인
- [ ] PC(데스크톱) 브라우저에서는 기존과 동일하게 카드 테두리/그림자/패딩이 정상적으로 보이는지 확인 (이번 변경으로 데스크톱 레이아웃이 깨지지 않아야 함)
- [ ] 주간 뷰(`WeekGrid`)에서 가로 스크롤 영역의 좌우 여백(`-mx-1`/`mx-0`)이 세로/가로모드에서 일관되는지 확인
- [ ] 일간 뷰(`DayView`)에서 좌우 패딩이 세로/가로모드에서 일관되는지 확인

## 테스트 우선순위

1. **1번** PWA 재설치 후 가로모드 회전 확인 — 이번 변경의 핵심 목적
2. **2번** 가로모드 달력 여백 확인 — 회전 후에도 세로모드와 동일한 레이아웃인지가 이번 추가 수정의 핵심
3. **N번** 세로모드/PC 회귀 확인 — 변경이 기존 동작에 영향 없는지 확인
