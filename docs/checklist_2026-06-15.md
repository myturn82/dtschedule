## 1. 개발 서버 무한 리로드 현상 수정

배경: 재기동 후 dev 서버가 계속 리로딩되는 현상이 이 PC에서만 심하게 발생.
이전 프로덕션 빌드가 등록한 PWA 서비스워커/캐시가 localhost에 남아 충돌을 일으키고,
`.omc/**` 디렉토리가 계속 갱신되며 Vite 워처를 자극하는 것이 원인으로 추정됨.

- [ ] 브라우저에서 `localhost:5173` 접속 후 **Ctrl+Shift+R**(하드 리프레시) 1회 수행
- [ ] 개발자도구 → Application → Service Workers에 등록된 SW가 없음(또는 자동 해제됨) 확인
- [ ] 이후 `npm run dev` 재기동 시 더 이상 자동 리로드가 반복되지 않음
- [ ] 코드 수정 시 HMR(핫 리로드)은 정상적으로 동작함 (전체 새로고침이 아닌 부분 갱신)

## 2. 스케줄 등록 모달 — 모드 무관 커스텀 필드 노출

배경: 관리자 콘솔 "입력 필드" 탭에서 등록한 커스텀 필드가 회원공유/회원개별 모드에서는
스케줄 등록 모달에 표시되지 않던 문제. 모드와 상관없이 등록된 커스텀 필드를 모두 표시하도록 수정.

- [ ] **회원공유/회원개별 모드** + 커스텀 필드 미등록 상태
  - 스케줄 등록 모달에 기존과 동일하게 추가 입력란 없음 (변화 없음)
- [ ] **회원공유/회원개별 모드** + 커스텀 필드 등록한 상태
  - 스케줄 등록 모달의 인원 추가 입력란 하단에 등록한 커스텀 필드(텍스트/선택)가 표시됨
  - 필수(`required`) 필드를 비워두면 "추가" 버튼이 비활성화되거나 에러 메시지 표시
  - 값 입력 후 등록 → 배정 목록에 "필드명: 값" 형태로 표시됨
  - 기존 배정 수정(연필 아이콘) 시 입력했던 커스텀 필드 값이 그대로 로드되어 수정 가능
- [ ] **비회원(직접입력) 모드**: 기존과 동일하게 동작 (커스텀 필드가 메인 입력 폼으로 사용됨)
- [ ] 관리자 콘솔 "입력 필드" 탭
  - 회원공유/회원개별 모드: "커스텀 필드 (N개)" 안내문 + "스케줄 등록 시 추가로 입력받을 정보를 설정합니다..." 설명 표시
  - 비회원 모드: 기존과 동일하게 "첫 번째 필드가 이름(성명) 필드로 사용됩니다" 안내 표시
