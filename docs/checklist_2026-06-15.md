## 7. 스케줄 등록/수정 모달(SlotEditModal) 디자인 개편

배경: 디자인 시안("Slot Edit Modal")을 반영해 모달 전체 레이아웃·스타일을 개편.
기존 기능(커스텀 필드, 회원/비회원 모드, 배정 추가·수정·삭제·잠금, 빈 슬롯 알림, 인원 초과 경고 등)은
모두 동일하게 동작해야 하며, 시각적 스타일만 변경됨.

- [ ] **헤더**: "관리자 등록 / 내 스케줄 등록 / 슬롯 상세 / 배정 수정" 등 상황별 뱃지(키커)가 올바르게 표시됨
- [ ] **헤더**: `N월 N일 (요일)` 제목과 시간/구분/역할명/인원 현황(`N/M명`)이 한 줄에 표시됨
  - 인원이 가득 찼을 때(`isFull`) 인원 현황 색상이 일간 뷰(`DayView`)의 "가득 참" 색상과 동일하게 초록색 계열로 표시됨
- [ ] **닫기 버튼**: 사각형 테두리 + X 아이콘으로 표시되고 정상 동작함
- [ ] **관리자 + 역할 분리(역할 2개 이상) 모드**: 역할 선택이 알약(pill) 버튼 형태로 표시되고, 선택한 역할에 강조 스타일 적용됨
- [ ] **일반 사용자 + 역할 분리 모드**: 자신의 역할이 알약 형태 뱃지로 표시됨(선택 불가)
- [ ] **기존 배정 목록**: 각 배정이 카드 형태로 표시되며
  - 이름 앞에 동그란 아바타(이름 첫 글자)가 표시됨
  - 본인 배정에는 "나" 뱃지가 표시되고 카드 배경이 강조색으로 옅게 표시됨
  - 잠금된 배정은 자물쇠 아이콘과 회색 배경으로 표시됨
  - 연락처/메모/커스텀 필드 값이 카드 하단에 칩(chip) 형태로 표시됨
  - 수정/삭제/고정/해제 버튼이 정상 동작함
- [ ] **입력 필드**: 포커스 시 테마 강조색(`--color-brand-primary`) 기준 포커스 링이 표시됨 (파란색 아님)
- [ ] **시간 세부 선택(있는 경우)**: 알약 버튼 형태로 표시되고, 선택 시 강조색으로 채워짐
- [ ] **회원 선택/본인 표시**: 아바타 + 이름 형태의 "신원 칩"으로 표시됨
- [ ] **하단 영역(푸터)**: 에러 메시지, "빈 슬롯 알림" 토글(관리자), 저장/취소 버튼이 모달 하단에 고정되어 본문 스크롤과 분리됨
- [ ] **모바일 화면**: 모달이 화면 하단에서 올라오는 바텀시트 형태로 표시됨 (둥근 위쪽 모서리)
- [ ] **PC 화면**: 모달이 화면 중앙에 카드 형태로 표시됨
- [ ] 월간 뷰(`ScheduleGrid`) / 주간 뷰(`WeekGrid`) / 일간 뷰(`DayView`) / 모바일 뷰(`MobileScheduleView`) 각각에서 셀 클릭 시 위 디자인이 동일하게 적용됨

## 8. 역할 분리 + 비회원/직접입력 모드 — 관리자 등록 시 "회원에서 선택" 추가

배경: 역할 분리(splitRoles)와 비회원/직접입력(타입니주) 모드를 함께 사용하는 조직에서,
관리자가 역할 알약 버튼을 선택해도 해당 역할에 등록된 회원을 고를 방법이 없고
매번 이름/연락처를 직접 입력해야 했던 문제.

- [ ] 관리자 등록 모달에서 역할(알약 버튼)을 선택하면, 입력란 위에 **"회원에서 선택 (선택)"** 드롭다운이 표시됨
  - 드롭다운에는 해당 역할에 등록되어 있고 이 슬롯에 아직 배정되지 않은 회원만 표시됨
  - 해당 역할에 선택 가능한 회원이 없으면 드롭다운 자체가 표시되지 않음 (기존과 동일하게 이름/연락처 직접 입력)
- [ ] 드롭다운에서 회원을 선택하면 이름 입력란(또는 커스텀 필드의 첫 번째 필드)에 해당 회원 이름이 자동으로 채워짐
- [ ] 자동으로 채워진 이름을 직접 수정하면 "회원에서 선택"이 다시 "-- 직접 입력 --"으로 풀림 (연동 해제)
- [ ] 연락처는 회원 선택 여부와 무관하게 계속 직접 입력해야 함 (기존과 동일)
- [ ] 역할 알약 버튼을 다른 역할로 전환하면 "회원에서 선택"이 "-- 직접 입력 --"으로 초기화되고, 새 역할에 등록된 회원 목록으로 갱신됨
- [ ] 등록 후 배정 목록에 정상적으로 표시됨 (회원 선택 여부와 관계없이 이름/연락처가 올바르게 저장됨)
- [ ] 기존 배정을 "수정"할 때도 이전에 "회원에서 선택"으로 등록된 항목이면 드롭다운에 해당 회원이 선택된 상태로 표시됨
- [ ] **회귀 확인**: 역할 분리가 없는 비회원/직접입력 조직에서는 "회원에서 선택" 드롭다운이 나타나지 않고 기존과 동일하게 동작함
- [ ] **회귀 확인**: 회원선택 모드(역할 분리 여부 무관)의 기존 "회원 선택" 동작은 변화 없음
- [ ] 빌드: `npx tsc -b` 통과 (확인 완료)

## 10. 회원(로그인) — 슬롯 클릭 시 즉시 배정

배경: 일반 회원이 로그인한 상태에서 빈 슬롯을 클릭하면, 모달을 열고 "회원 선택"/"나" 카드를
확인한 뒤 "저장"을 눌러야 등록되던 흐름을 단순화. 커스텀 필드 입력이 필요 없는 조직에서는
클릭 한 번으로 자신을 즉시 배정하도록 변경.

- [ ] **일반 회원 로그인 상태에서 빈 슬롯(자리 여유 있음) 클릭** → 모달 없이 즉시 "등록되었습니다" 토스트가 뜨고 본인 이름으로 배정됨
- [ ] **근무시간 세부선택(10시/11시/10~12시 등)이 있는 슬롯**에서도 클릭 한 번으로 즉시 배정되며, 세부시간은 기본값(마지막 옵션, 예: "10~12시")으로 자동 적용됨
- [ ] **정원이 2자리 이상 남은 슬롯**에서도 (기존엔 1자리 남았을 때만 즉시배정) 클릭 한 번으로 즉시 배정됨
- [ ] **이미 본인이 배정된 슬롯을 다시 클릭** → 즉시 취소되지 않고 모달이 열림 (기존과 동일, 수정/삭제 가능)
- [ ] **비회원/직접입력 모드 + 커스텀 필드 등록된 조직**: 클릭 시 즉시 배정되지 않고 모달이 열림 (이름/연락처 등 입력 필요, 변화 없음)
- [ ] **비회원/직접입력 모드 + 커스텀 필드 0개인 조직(역할분리 포함)**: 클릭 한 번으로 본인 이름으로 즉시 배정됨
- [ ] **회원개별 모드**: 변화 없음 (모달을 통한 등록 유지)
- [ ] **역할 분리 모드**: 본인 역할의 슬롯에서만 즉시 배정 동작 확인 (다른 역할 슬롯 클릭 시 기존과 동일하게 무시됨)
- [ ] 정원 마감/휴관/휴게시간/잠금된 슬롯 클릭 시 즉시배정 되지 않고 기존과 동일하게 동작
- [ ] 월간 뷰(`ScheduleGrid`) / 주간 뷰(`WeekGrid`) / 일간 뷰(`DayView`) 각각에서 동일하게 동작
- [ ] **관리자**: 클릭 시 즉시 배정되지 않고 항상 모달이 열림 (변화 없음)
- [ ] 빌드: `npx tsc -b` 통과 (확인 완료)

## 11. 이미 등록된 내 스케줄 클릭 시 "중복 등록" 오류 수정

배경: (10번 즉시 배정 기능 적용 후) 회원이 이미 자신이 배정된 슬롯을 다시 클릭하면 모달이 열리는데,
이때 기존 배정 카드 아래에 "새로 추가" 입력란도 함께 표시되어, 거기서 메모만 수정하고 "저장"을 누르면
같은 이름으로 새 배정을 시도해 "이미 같은 회원이 배정되어 있습니다" 오류가 발생하던 문제.
이제 본인 명의의 배정이 이미 있으면 "새로 추가" 입력란 대신 "내 스케줄 수정" 버튼이 표시되고,
이를 클릭하면 기존 수정 모드(입력값 미리 채워짐)로 진입한다. 해당 배정 카드의 "수정" 버튼은
"내 스케줄 수정" 버튼으로 대체되어 숨겨진다("삭제" 버튼은 유지).

- [ ] **회원공유/비회원(직접입력, 커스텀필드 없음) 모드**: 이미 본인이 배정된 슬롯 클릭
  - 모달 상단에 기존 배정 카드가 표시되고, 해당 카드에는 "수정" 버튼이 보이지 않음("삭제"는 유지)
  - 그 아래 "새로 추가" 입력란 대신 "이미 등록된 내 스케줄입니다. 아래 버튼으로 수정할 수 있습니다." 안내 문구 표시
  - 하단 버튼이 "내 스케줄 수정" / "닫기"로 표시됨
- [ ] **"내 스케줄 수정" 클릭** → 기존 수정 모드(메모/근무시간 등 입력값이 현재 값으로 미리 채워짐)로 전환되고, 하단 버튼이 "수정 완료" / "취소"로 바뀜
- [ ] 수정 모드에서 메모 수정 후 "수정 완료" 클릭 → 정상 저장됨 (중복 등록 오류 없음)
- [ ] "취소" 클릭 → 다시 "내 스케줄 수정" 버튼이 있는 안내 화면으로 복귀
- [ ] **본인 배정이 없는 빈 슬롯** 클릭 시: 기존과 동일하게 "새로 추가" 입력란과 "저장" 버튼이 정상 표시됨 (변화 없음)
- [ ] **관리자**: 다른 회원의 배정이 있는 슬롯을 클릭해도 기존과 동일하게 "수정"/"삭제" 버튼과 "회원 선택" 입력란이 정상 표시됨 (변화 없음)
- [ ] **비회원/직접입력 모드 + 커스텀 필드 등록된 조직**: 변화 없음 (동적 필드 입력 폼 그대로 유지)
- [ ] **고정(잠금)된 본인 배정**: 기존과 동일하게 "고정된 항목은 수정할 수 없습니다" 안내만 표시 (이번 변경의 영향 없음)
- [ ] 월간 뷰(`ScheduleGrid`) / 주간 뷰(`WeekGrid`) / 일간 뷰(`DayView`) / 모바일 뷰 각각에서 동일하게 동작 (SlotEditModal 공용이므로 자동 반영)
- [ ] 빌드: `npx tsc -b` 통과 (확인 완료)
