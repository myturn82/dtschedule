## 1. 배정 건 Hold(잠금) 기능

배경: 회원이 일정에 등록한 후, 관리자가 "고정(hold)"을 걸면 그 즉시 등록한 회원 본인, 조직관리자, **슈퍼관리자를 포함한 누구도** 해당 건을 수정/삭제/메모 변경할 수 없다. 오직 슈퍼관리자의 "해제"만 가능하며, 해제 후에만 기존 권한 규칙대로 수정/삭제가 재개된다.

- [ ] 일반 회원으로 로그인 → 일정 슬롯에 본인을 등록
- [ ] 조직관리자로 로그인 → 해당 항목에 "고정" 버튼(자물쇠 아이콘)이 보이고 클릭 시 정상 적용됨
- [ ] 고정 후 해당 항목에 자물쇠 아이콘이 표시됨 (SlotEditModal 목록, 스케줄 그리드 셀)
- [ ] 일반 회원(등록자 본인)으로 다시 확인 → 수정/삭제 버튼이 보이지 않고 메모 등 어떤 항목도 변경 불가
- [ ] 조직관리자로 다시 확인 → 고정된 항목에 수정/삭제/해제 버튼이 보이지 않음
- [ ] 슈퍼관리자로 로그인 → 고정된 항목에는 수정/삭제 버튼이 보이지 않고 "해제" 버튼(열린 자물쇠 아이콘)만 보임
- [ ] **[DB 확인]** 잠긴 행에 대해 API/콘솔로 직접 update 시도 시 `assignment is locked` 에러로 차단됨 (수정/삭제 모두)
- [ ] 슈퍼관리자가 "해제" 클릭 → 고정 해제 후 조직관리자/회원이 다시 수정/삭제 가능해짐

## 6. 회원탈퇴(계정 삭제) 시 Edge Function 오류 수정

배경: `assignments.user_id`는 `profiles(id)`를 `ON DELETE SET NULL`로 참조하고, `profiles.id`는 `auth.users`를 `ON DELETE CASCADE`로 참조한다.
회원탈퇴(`delete-account` Edge Function → `auth.admin.deleteUser`) 시 profile 삭제가 cascade되면서
해당 사용자의 `assignments.user_id`를 `NULL`로 변경하는 UPDATE가 발생하는데, 이 UPDATE도
`check_assignment_lock_update` 트리거(잠긴 배정 건 수정 차단)에 걸려 잠긴(hold) 배정을 가진 계정은
탈퇴 자체가 `Edge Function returned a non-2xx status code` 에러로 실패하던 문제를 수정 (`045_assignment_lock_user_delete_fix.sql`).

- [ ] 본인 배정 건이 관리자에 의해 "고정(hold)"된 일반 회원으로 로그인 → 마이페이지에서 "회원탈퇴" 클릭 → 정상적으로 탈퇴 처리됨 (Edge Function 오류 없음)
- [ ] 탈퇴 후 해당 배정 건은 그대로 남아있고 `assignments.user_id`만 `NULL`로 변경됨, `member_name`/`is_locked` 등 다른 필드는 유지됨 (스케줄 화면에 "삭제됨" 표시)
- [ ] 고정되지 않은 일반 배정을 가진 회원의 탈퇴도 기존과 동일하게 정상 동작
- [ ] 조직 탈퇴(특정 tenant만 탈퇴, `tenant_id` 전달)도 정상 동작 (영향 없음 확인)
- [ ] **[DB 확인]** 탈퇴 후 `auth.users`/`profiles`에서 해당 사용자가 삭제되고, `assignments`의 잠긴 행은 `is_locked = true` 그대로 유지됨

## 7. 전체 회원탈퇴 후에도 예전 배정 건 "삭제됨" 표시

- [ ] 일반 회원으로 등록 → 해당 계정으로 "회원탈퇴"(전체 탈퇴) 진행

- [ ] 모바일 뷰에서도 동일하게 표시됨
- [ ] 동일 이메일로 재가입 후 새 계정으로 같은 슬롯에 새로 등록 → 새 등록 건은 "삭제됨" 표시 없이 정상 표시되고, 예전 탈퇴 건과는 별개로 구분됨

## 9. 다른 조직에서 동일 이름으로 등록 시 unique 제약 충돌(409) 수정

배경: `unique_member_assignment` 인덱스가 `(year, month, day, time_slot, member_name)` 기준으로
생성되어 있어 `tenant_id`가 빠져 있었다 (013에서 추가했으나 030의 컬럼명 변경 작업에서 인덱스를
재생성하며 누락된 회귀). 그 결과 한 회원이 다른 조직(tenant)에 새로 가입한 뒤 같은 날짜·시간대에
등록을 시도하면 다른 조직의 동명 배정과 충돌해 `duplicate key value violates unique constraint
"unique_member_assignment"` (409) 오류가 발생했다. `047_fix_unique_member_assignment_tenant.sql`로
인덱스를 `(tenant_id, year, month, day, time_slot, member_name)` 기준으로 재생성하여 조직별로
범위를 분리했다.

- [ ] 동일한 이름의 회원이 A조직에서 특정 날짜·시간대에 등록된 상태에서, 같은 사용자(또는 동명
  사용자)가 B조직에 가입 후 같은 날짜·시간대에 등록 → 409 오류 없이 정상 등록됨
- [ ] 같은 조직(tenant) 내에서 동일한 날짜·시간대·이름으로 중복 등록 시에는 여전히 차단됨 (기존 동작 유지)
- [ ] **[DB 확인]** `unique_member_assignment` 인덱스가 `(tenant_id, year, month, day, time_slot,
  member_name)` 기준으로 생성되어 있음

## 10. "초기화"(스케줄 일괄 삭제) 시 잠긴 배정이 있으면 전체 실패하던 문제 수정

배경: 043-strict 정책("잠긴 행은 누구도 삭제 불가, 슈퍼관리자도 예외 없음")이 운영에 반영된 후,
"초기화"(월/주/일 일괄 삭제) 기능이 대상 범위에 고정(hold)된 배정이 하나라도 있으면
DELETE 문 전체가 `assignment is locked` 에러(400 Bad Request)로 실패하던 문제를 수정.
`clearAssignments`의 DELETE 쿼리에 `is_locked = false` 조건을 추가해, 고정되지 않은 배정만
삭제되고 고정된 배정은 그대로 유지되도록 했다. 고정된 배정이 존재했던 경우, 삭제 완료 후
"고정된 배정 N건은 삭제되지 않고 유지됩니다." 안내가 표시된다.

- [ ] 월 뷰에서 일부 배정이 고정(hold)된 상태로 "초기화" 클릭 → 에러 없이 정상 처리되고,
  "고정된 배정 N건은 삭제되지 않고 유지됩니다." 안내가 표시됨
- [ ] 초기화 후 고정되지 않은 배정만 삭제되고, 고정된 배정은 화면에 그대로 남아있음
- [ ] 주 뷰에서 월 경계를 넘는 주(인접 월 포함)에 고정된 배정이 있을 때 "초기화" 적용 시
  양쪽 월 모두 동일하게 동작함 (고정 건 유지 + 안내 메시지)
- [ ] 일 뷰에서도 동일하게 동작함
- [ ] 고정된 배정이 전혀 없는 범위에서 "초기화" 클릭 시 기존과 동일하게 전체 삭제되고
  추가 안내 메시지는 표시되지 않음
- [ ] **[DB 확인]** 초기화 후 `assignments` 테이블에서 `is_locked = true`인 행은 삭제되지 않고
  남아있음, `is_locked = false`인 행만 삭제됨

## 11. 월간 스케줄 엑셀 다운로드 기능 추가

배경: 메뉴(상단 함수 메뉴)에 "엑셀 다운로드" 항목을 추가하여, 현재 보고 있는 월의 스케줄을
`.xlsx` 파일로 내보낼 수 있도록 했다 (`src/utils/exportSchedule.ts`의
`exportMonthScheduleToExcel`, `exceljs` 라이브러리 사용). 첫 행에 "{조직명} {연}년 {월}월 스케줄"
제목(병합 셀), 둘째 행에 시간대 + 날짜별 헤더("M/D (요일)"), 이후 행은 시간대별로 각 날짜의
배정 현황을 표시한다. 휴관일은 한글 공휴일명 또는 "휴관", 휴게시간은 "휴게", 휴무는 "휴무"로
표시되며, 배정 셀에는 회원명(스플릿 모드 시 역할명 포함)·메모·고객정보·`[고정]`(잠긴 배정)·
`[삭제됨]`(탈퇴 회원) 표시가 줄바꿈으로 나열된다. `displayAssignmentFilter`를 적용해
'회원개별' 모드에서 일반 회원에게는 본인 배정만 노출되도록 기존 화면과 동일한 표시 권한을 따른다.

- [ ] 메뉴(함수 메뉴)에 "엑셀 다운로드" 항목이 보이고 클릭 시 `.xlsx` 파일이 다운로드됨
- [ ] 파일명이 `{조직명}_{연}-{월}_스케줄.xlsx` 형식으로 생성됨
- [ ] 다운로드한 파일을 엑셀/한글 스프레드시트 프로그램에서 열었을 때:
  - [ ] 첫 행에 "{조직명} {연}년 {월}월 스케줄" 제목이 날짜 수에 맞게 병합되어 표시됨
  - [ ] 둘째 행에 "시간대" + 각 날짜("M/D (요일)") 헤더가 표시됨
  - [ ] 시간대별 행에 해당 날짜·시간대의 배정자 이름이 정상적으로 표시됨
  - [ ] 공휴일/휴관일에는 공휴일명 또는 "휴관"으로 표시됨
  - [ ] 휴게시간 슬롯은 "휴게"로 표시됨
  - [ ] 휴무 슬롯은 "휴무"로 표시됨
  - [ ] 고정(hold)된 배정에는 `[고정]` 표시가 포함됨
  - [ ] 탈퇴한 회원의 배정에는 `[삭제됨]` 표시가 포함됨
  - [ ] 스플릿 모드(역할별 분리) 조직에서는 배정에 역할명이 함께 표시됨
- [ ] '회원개별' 모드에서 일반 회원으로 다운로드 시, 본인이 화면에서 볼 수 있는 배정만 엑셀에도
  동일하게 표시됨 (다른 회원의 배정은 노출되지 않음)
- [ ] 주/일/모바일 뷰 등 다른 화면에는 영향 없음 (엑셀 다운로드는 월간 스케줄 기준)

## 테스트 우선순위

1. **1번** Hold 기능의 핵심 시나리오 — 회원/조직관리자가 고정된 건을 수정/삭제할 수 없는지, 슈퍼관리자만 예외로 처리 가능한지 확인
2. **2번** 월별 일괄 고정/해제 — 권한별 메뉴 노출, 대상 없음 처리, 월 경계(주 뷰) 처리 확인
3. **3번** 잠긴 항목의 메모 저장 버그 수정 확인
4. **4번** 미배정 슬롯 잠금 — 빈 슬롯에 자물쇠 표시 및 신규 등록 차단/해제 확인
5. **5번** 회원탈퇴 시 잠긴 배정 건이 있어도 Edge Function 오류 없이 정상 탈퇴되는지 확인
6. **6번** 전체 탈퇴 후 예전 배정 건이 월/주/일/모바일 뷰 모두에서 "삭제됨"으로 표시되는지 확인
7. **7번** 기존 등록/수정/삭제 흐름이 깨지지 않았는지 회귀 확인
8. **9번** 다른 조직에 동일 이름으로 등록 시 409(unique 제약) 오류 없이 정상 등록되는지 확인
9. **10번** "초기화" 시 고정된 배정이 있어도 에러 없이 정상 처리되고 고정 건이 보존되는지 확인
10. **11번** 월간 스케줄 엑셀 다운로드 기능 — 메뉴 노출, 파일 다운로드, 내용(제목/헤더/배정/휴관/고정/삭제됨 표시) 정상 확인
