## 1. 회원가입 시 조직 선택 흐름 수정 (카카오 + 이메일 공통 점검)

배경: 카카오 탭 클릭 시 조직 선택 없이 바로 OAuth로 진행되어 복귀 후 PendingPage로 튕겼다.
조직 선택 위자드를 먼저 표시하도록 흐름을 수정했다. 이메일 가입도 동일 위자드를 공유하므로 함께 점검한다.

### 카카오 가입

- [ ] 회원가입 탭에서 "카카오로 계속하기" 클릭 → 조직 선택 위자드(choice 단계)가 먼저 표시되는가
- [ ] "기존 조직에 가입하기" 선택 → 조직 목록에서 선택 후 "카카오로 가입 신청하기" 클릭 → 카카오 OAuth 진행 → 복귀 후 승인 대기(PendingPage) 표시되는가
- [ ] "내 서비스 시작하기" 선택 → "카카오로 계속하기" 클릭 → OAuth 복귀 후 `/setup` 또는 서비스 시작 화면으로 이동하는가
- [ ] 이미 카카오 계정이 연동된 상태로 재로그인 시 정상 동작하는가

### 이메일 가입

- [ ] 회원가입 탭에서 "이메일로 계속하기" 클릭 → 이메일 입력 → 이름/비밀번호 위자드가 순서대로 표시되는가
- [ ] 위자드 마지막 단계(choice)에서 "기존 조직에 가입하기" 선택 → 조직 검색·선택 → "가입 신청하기" 클릭 → 승인 대기(PendingPage) 화면이 표시되는가
- [ ] "내 서비스 시작하기" 선택 → 서비스 이름·전화번호 입력 → "시작하기" 클릭 → `/setup` 화면으로 이동하는가
- [ ] 이미 가입된 이메일 입력 시 "이미 가입된 이메일입니다" 오류 후 로그인 탭/비밀번호 입력으로 전환되는가

## 2. 미운영 일 CLOSE 텍스트 제거 (줄무늬 배경만 표시)

배경: 운영하지 않는 날짜 셀에 "CLOSE" 텍스트가 표시되어 불필요한 노이즈가 있었다.
줄무늬 배경만 표시하도록 변경했다.

- [ ] 월간 뷰에서 미운영 날짜 셀에 "CLOSE" 텍스트가 사라지고 줄무늬 배경만 표시되는가
- [ ] 미운영 날짜 셀에서 줄무늬 배경이 정상적으로 렌더링되는가
- [ ] 운영 날짜 셀은 영향받지 않고 기존대로 표시되는가

---

## 3. 연중무휴 조직에서 공휴일·일요일도 운영으로 표시

배경: `is_open=true` 규칙이 있는 연중무휴 조직에서도 공휴일(광복절·추석 등)이
휴관으로 잘못 표시되는 문제가 있었다. override가 없을 때 holiday 체크를 건너뛰도록 수정했다.

- [ ] 연중무휴(`is_open=true`) 조직 달력에서 공휴일(예: 8/15 광복절)이 운영 상태로 표시되는가
- [ ] 연중무휴 조직에서 일요일도 운영 상태로 표시되는가
- [ ] 연중무휴 조직에서 공휴일에 명시적으로 `is_open=false` dateOverride가 있으면 정상적으로 휴관으로 표시되는가
- [ ] 일반(비연중무휴) 조직에서 공휴일이 기존대로 휴관으로 표시되는가
