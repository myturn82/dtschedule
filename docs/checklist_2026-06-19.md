## 1. 엑셀모드 — 분리된 열(역할/회원·50+) 단위 복사·붙여넣기

배경: 역할 분리 모드나 회원/50+ 분리 모드에서 한 칸을 선택하면 그 시간대의 모든 열이
한꺼번에 선택·복사되던 문제를 수정. 선택 좌표에 `colIdx`(열 인덱스)를 추가해
`day × slotIdx × colIdx` 3차원 박스로 확장. 설계 문서: `docs/superpowers/specs/2026-06-18-excel-mode-column-select-design.md`

- [ ] 역할 분리 모드: 역할 A 열의 한 칸만 선택(파란 음영이 그 역할 열에만 표시되는지 확인) → 다른 날짜의 같은 역할 A 칸을 클릭(두 번째 클릭으로 범위 완성) → 복사 → 다른 위치에 붙여넣기 → **역할 A 열에만** 들어가고 역할 B/C 열은 그대로인지 확인
- [ ] 역할 분리 모드: 역할 A~C 열 범위로 선택(역할 A 칸 클릭 후 역할 C 칸 클릭) → 복사 → 역할 B를 시작 열로 잡고 붙여넣기 → B, C 순서로 들어가고 그 이상 역할이 없으면 조용히 건너뛰는지 확인
- [ ] 회원/50+ 분리 모드: 50+ 열만 선택 → 복사 → 평일 50+ 열에 붙여넣기 정상 동작 확인
- [ ] 회원/50+ 분리 모드: 50+ 열 데이터를 복사한 뒤, **토요일** 위치에 붙여넣기 시도 → 에러 없이 조용히 건너뛰는지 확인 (토요일은 50+ 열이 없으므로 화면에 안 보이는 데이터가 생기면 안 됨)
- [ ] 회원/50+ 분리 모드: 회원(vol) 열에서 복사한 데이터를 50+ 위치에 붙여넣기 → 붙여넣은 데이터가 50+ 열에 정상적으로 보이는지 확인 (member_type이 타겟 열 기준으로 바뀌어야 함)
- [ ] **(버그 수정 확인)** 같은 요일·같은 시간대에서 역할 A 칸을 복사한 뒤, 곧바로 같은 요일·시간대의 역할 B 칸을 클릭해 붙여넣기 → B에 정상적으로 들어가는지 확인 (이전에는 A~B 범위로 잘못 묶여 A에 다시 붙여넣기를 시도하면서 B에는 아무 일도 안 일어나는 버그가 있었음)
- [ ] **(채우기 동작 확인)** 셀 1개만 복사 → 붙여넣을 대상으로 여러 칸(여러 날짜/슬롯/열)을 범위로 선택 → 붙여넣기 → **선택한 모든 칸**에 동일한 내용이 들어가는지 확인 (이전에는 선택 영역의 첫 칸 하나에만 들어갔음)
- [ ] 위 채우기 결과로 들어간 각 칸의 시간대(타임슬롯)가 **그 칸 자신의 시간대**로 정확히 표시되는지 확인 (복사한 원본 시간대가 아니라 붙여넣은 위치 기준)
- [ ] **(버그 수정 확인)** 2시간 슬롯(예: 10-12시)에서 "전체 담당"으로 등록된 셀을 복사해서 **다른 시간대의 2시간 슬롯**(예: 14-16시)에 붙여넣기 → 등록된 회원 정보 옆에 표시되는 시간이 "10~12시"가 아니라 붙여넣은 슬롯 기준 "14~16시"로 보이는지 확인 (이전에는 복사한 원본 슬롯의 시각이 그대로 남아있었음)
- [ ] 2시간 슬롯에서 "전반만 담당"/"후반만 담당"으로 등록된 셀도 복사 후 다른 2시간 슬롯에 붙여넣었을 때 그 슬롯의 전반/후반 시각으로 정확히 바뀌는지 확인
- [ ] **(버그 수정 확인)** 1시간 슬롯(예: 13-14시, 부분 시각 구분 없음)에서 등록된 셀을 복사해서 **2시간 슬롯**(예: 10-12시)에 붙여넣기 → "전체 담당"(10~12시)으로 표시되는지 확인 (이전에는 원본에 시각 정보가 없어서 그대로 비어 있었음)
- [ ] 2시간 슬롯 → 1시간 슬롯으로 붙여넣기는 여전히 시각 표시가 없는 게 맞는지 확인 (1시간 슬롯은 원래 부분 시각 구분이 없는 정상 동작)

## 2. 엑셀모드 — PC·모바일 선택 규칙 분리 + 액션 바

배경: 범위 선택이 Shift 키(물리 키보드)에 의존하고, 복사/붙여넣기가 Ctrl+C/V 키보드 단축키로만
실행되어 모바일에서 엑셀모드가 동작하지 않던 문제를 수정. 처음에는 PC·모바일에 동일한
"두 번 탭 = 범위 선택" 규칙을 적용했으나, PC에서 복사 직후 클릭이 의도와 다르게 범위를
확장해버리는 문제가 발견되어 **PC는 기존 방식(클릭=항상 새 선택, Shift+클릭=확장)으로 되돌리고
모바일만 두 번 탭 방식을 사용**하도록 분리했다(`pointer: coarse`로 기기 판별). 하단 고정
액션 바(복사/붙여넣기/선택해제 버튼)도 함께 추가.

- [ ] PC: 클릭할 때마다 항상 그 칸 하나만 새로 선택되는지 확인 (이전 선택과 합쳐지지 않음) — 기존 방식 그대로
- [ ] PC: Shift+클릭으로 끝점을 계속 확장하는 동작이 회귀 없이 동작하는지 확인
- [ ] PC: Ctrl+C / Ctrl+V / Esc 단축키가 모두 회귀 없이 동작하는지 확인
- [ ] **(버그 수정 확인)** PC: 한 칸 선택 → Ctrl+C → (다른 칸을 클릭하지 않고) 바로 Ctrl+V → 같은 자리에 정상적으로 다시 붙여넣기 동작하는지 확인 (선택을 다른 칸으로 옮기지 않은 키보드 전용 흐름)
- [ ] 모바일(터치, 실기기 또는 Chrome DevTools 모바일 에뮬레이션): 첫 칸 탭 → 다른 칸 탭 → 그 사이가 범위로 자동 완성되는지 확인
- [ ] 모바일: 범위가 완성된 상태에서 또 다른 칸을 탭하면 그 칸부터 **새 선택**이 시작되는지 확인
- [ ] 화면 하단에 엑셀모드 선택 중 "📋 복사 / 📥 붙여넣기 / ✕ 선택해제" 액션 바가 뜨는지 확인 (PC·모바일 공통)
- [ ] 복사를 누른 직후에도 액션 바가 사라지지 않고 "붙여넣기"/"선택해제"가 계속 보이는지 확인 (선택은 비워지지만 복사 내용은 유지되어야 함)
- [ ] 선택 영역이 없을 때 "복사" 버튼이 비활성화되는지, 복사된 내용이 없거나 권한이 없을 때 "붙여넣기" 버튼이 비활성화되는지 확인
- [ ] 모바일에서 액션 바 버튼으로 복사/붙여넣기/선택해제가 정상 동작하는지 확인
- [ ] "선택해제" 버튼을 누르면 액션 바가 사라지고 파란 음영도 없어지는지 확인

## 3. 엑셀모드 — 커스텀 필드 복사 + 되돌리기(Ctrl+Z)

배경: 복사·붙여넣기 시 비회원(자유입력) 모드의 커스텀 필드 값(`extra_data`)과 고객명/연락처가
복사되지 않던 것을 수정. 붙여넣기로 새로 생성된 배정을 되돌릴 수 있는 기능을 추가
(PC: Ctrl+Z, 모바일: 액션 바의 "↩️ 되돌리기" 버튼). **엑셀모드에 진입한 시점부터 누적되는 스택
방식**이라, 여러 번 붙여넣은 뒤 되돌리기를 반복 호출하면 엑셀모드를 켠 시점까지 거슬러 되돌릴 수
있다(redo는 없음). 엑셀모드를 껐다 켜면 스택이 비워진다.

- [ ] 비회원(자유입력) 모드: 커스텀 필드 값이 입력된 셀을 복사 → 붙여넣기 → 붙여넣은 셀에도 동일한 커스텀 필드 값이 보이는지 확인
- [ ] 비회원 모드: 고객명/연락처가 있는 셀을 복사 → 붙여넣기 → 고객명/연락처가 정상적으로 복사되는지 확인
- [ ] PC: 붙여넣기 직후 Ctrl+Z → 방금 붙여넣은 항목들이 모두 삭제(원상복구)되는지 확인
- [ ] **(스택 확인)** 붙여넣기를 3번 연속 다른 위치에 한 뒤 Ctrl+Z를 3번 누르면, 매번 가장 최근 붙여넣기부터 역순으로 하나씩 되돌려지고 세 번째에는 엑셀모드 진입 시점(붙여넣기 이전) 상태로 돌아오는지 확인
- [ ] 되돌릴 게 더 없을 때 Ctrl+Z를 눌러도 에러 없이 아무 일도 안 일어나는지 확인
- [ ] 모바일: 붙여넣기 후 액션 바의 "↩️ 되돌리기" 버튼이 활성화되는지, 여러 번 눌러서 누적 붙여넣기를 차례로 되돌릴 수 있는지 확인
- [ ] 되돌릴 항목이 없을 때(아직 붙여넣은 적이 없거나, 모두 되돌린 후) "되돌리기" 버튼이 비활성화되는지 확인
- [ ] 엑셀모드를 껐다가 다시 켜면 이전 되돌리기 스택이 사라지는지 확인 (되돌리기 버튼이 비활성화 상태로 시작)
